# AI Engineering Learning

A personal knowledge base for building real capability in software engineering, machine learning systems, AI evaluations, LLM systems, agents, and production AI.

This repo preserves and organizes what I learn across courses, books, papers, work, and hands-on projects. It is a working notebook, not a portfolio site: notes get promoted into reusable playbooks, terminology into a glossary, and demonstrated skills into a skills matrix.

## Start here

- [LEARNING_HUB.md](LEARNING_HUB.md) — the front door. Current focus, active resources, and links to everything else.
- [ROADMAP.md](ROADMAP.md) — where I'm going and in what order.
- [SKILLS_MATRIX.md](SKILLS_MATRIX.md) — self-assessed skill levels with evidence.
- [GLOSSARY.md](GLOSSARY.md) — terms and definitions.
- [PORTFOLIO.md](PORTFOLIO.md) — completed projects and artifacts.

## Layout

```
courses/     Raw notes per course, one folder each
playbooks/   Reusable, distilled how-to guides
projects/    Write-ups of hands-on work
templates/   Reusable note templates
```

## How I use this

1. Take raw notes in the relevant `courses/` folder while learning.
2. Promote anything reusable into `playbooks/`.
3. Add new terminology to `GLOSSARY.md`.
4. Update `SKILLS_MATRIX.md` only when I have real evidence of progress.
5. Record finished work in `PORTFOLIO.md`.

See [CLAUDE.md](CLAUDE.md) for the working rules I follow (and that Claude Code follows) when maintaining this repo.
